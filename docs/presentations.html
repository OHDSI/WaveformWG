Related Publications

CHoRUS Multimodal Linkage SOP

Publication: CHoRUS Bridge2AI Standards Office

Description: Standard Operating Procedure for linking multimodal data (including waveforms) with electronic health records. This SOP introduced the waveform_registry table concept, which was later expanded into the 4-table Waveform Extension.

Link: https://chorus-ai.github.io/Chorus_SOP/docs/Multimodal-Linkage/

Community Contributions

We encourage community members to share their work using the Waveform Extension:

  • Implementation case studies: Share your ETL approach and lessons learned
  • Tool development: Present utilities, converters, or analysis pipelines
  • Research applications: Showcase studies using integrated waveform + EHR data
  • Validation studies: Report data quality metrics and conformance testing

Submit presentation proposals via: - GitHub: Open an issue with the “presentation” label - Email: houghtaling@ohdsi.org - Teams: Post in the Waveform Working Group channel
